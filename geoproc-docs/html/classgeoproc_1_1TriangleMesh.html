Detailed Description

Implementation of a triangular mesh.

It contains the vertices of a triangular mesh (vertices) which are related using indices (triangles), hence following the usual implementation.

It also stores normal vectors per triangle (normal_vectors), angles and area of each triangle (angles, areas), and implements the corner table data structure (see corners, opposite_corners).

Finally, it holds other interesting information about the structure of the mesh, like boundaries (see boundary_edges, boundaries).

Adds the connectivity information to the mesh.

The contents of tris are the vertex indices of the triangles of the mesh. That is, face f has vertices vertices[3*f], vertices[3*f + 1], vertices[3*f + 2].

◆ boundary_edges

std::vector<int> geoproc::TriangleMesh::boundary_edges
protected

Boundary edges of the mesh.

The hard boundary is composed by pairs of vertices (each identified with a vertex index).

The hard is boundary is defined as those edges ({vi,vj}) of those triangles ({vi,vj,vk}) such that vk has no opposite corner. All vi, vj, vk are valid vertex indices.

Boundary vertices are collected into this container when the neighbourhood data is made (see make_neighbourhood_data). In order to make all boundary lists call make_boundaries, which will fill the member boundaries.

Edges per triangle.

In the DCEL, each triangle has exactly three edges incident to it, that is, three edges such that their right or left face is that triangle. Formally,

◆ opposite_corners

std::vector<int> geoproc::TriangleMesh::opposite_corners
protected

The set of opposite corners.

This table stores for every corner its opposite corner. Therefore, we can access the vertex opposite to another vertex (with respect to an edge).

In general, opposite_corners[i] = j iff corner j is the opposite corner of i.

In some cases this is not true, since the "opposite" corner is found by crossing a boundary. Such opposite is given as a negative value, but less than the number of triangles in absolute value if make_neighbourhood_data was called with a 'true' value. In this case, we need to consider orientations: take its opposite edge and the 'right' shared vertex (when placing the corner 'on top of' the opposite edge), now take the other boundary edge incident to the 'right' edge and, finally, this edge's opposite corner. The corner found is defined to be opposite of the starting corner. Notice that this is defined for counterclockwise traversals on the one-ring neighbourhood of the 'right' vertex.

◆ triangles

std::vector<int> geoproc::TriangleMesh::triangles
protected

The set of triangles of the mesh.

Every three consecutive values starting at a position multiple of 3 contain the indexes of the vertices (stored in vertices) of the triangle.

Then, the i-th triangle has vertices triangles[i], triangles[i+1], triangles[i+2].

Just to make it clear: triangles contains indices pointing to values in vertices.
